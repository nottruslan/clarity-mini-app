# Исправление ошибки деплоя Netlify

## Ошибка деплоя

Ошибка: "Failed retrieving extensions for site... fetch failed"

Это проблема с авторизацией Netlify CLI, но **не критична** для автоматического деплоя через Git.

## Что сделано

1. ✅ Файл `_headers` создан в `public/`
2. ✅ Файл `_headers` скопирован в `dist/`
3. ✅ Файлы добавлены в git и запушены

## Что дальше

Netlify автоматически задеплоит изменения через Git integration (не через CLI).

**Проверьте:**
1. Netlify Dashboard → **Deploys**
2. Должен появиться новый деплой (может занять 1-2 минуты)
3. Если деплой успешен - файл `_headers` будет применен

## Если деплой не работает

Ошибка авторизации Netlify CLI не влияет на автоматический деплой через Git. 

**Если деплой все еще не работает:**
1. Проверьте настройки Git integration в Netlify Dashboard
2. Убедитесь, что репозиторий правильно подключен
3. Проверьте, что последний коммит запушен в правильную ветку

## Важно

Файл `_headers` должен быть в `public/` (Netlify автоматически скопирует его) ИЛИ в `dist/` (чтобы точно попал в деплой). Я добавил его в оба места для надежности.
