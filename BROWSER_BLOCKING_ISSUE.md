# Проблема: Браузер не отправляет запросы

## Критическая проблема

В Network tab для JS и CSS файлов:
- **Отправлено байтов: 0Б**
- **Получено байтов: 0Б**
- **Нет заголовков запросов**
- **Нет заголовков ответов**

**Это означает:** Браузер блокирует запросы ДО их отправки. Запросы даже не начинаются.

## Возможные причины:

### 1. Mixed Content (HTTP/HTTPS конфликт)

**Проблема:** Если главная страница загружается по HTTPS, а ресурсы пытаются загрузиться по HTTP, браузер блокирует их.

**Проверка:**
1. В Network tab проверьте запрос к `claritybot.ru` (document)
2. Перейдите на вкладку **Headers**
3. Проверьте URL - должен быть `https://claritybot.ru/`
4. Проверьте, что в `index.html` все пути начинаются с `/` (относительные) или `https://`

### 2. Content Security Policy (CSP)

**Проблема:** CSP заголовки могут блокировать загрузку скриптов и стилей.

**Проверка:**
1. В Network tab кликните на запрос к `claritybot.ru` (document)
2. Перейдите на вкладку **Headers**
3. Проверьте **Response Headers** на наличие:
   - `Content-Security-Policy`
   - `X-Content-Security-Policy`
4. Если есть - проверьте, разрешают ли они загрузку из `/assets/*`

**Решение:**
- Если CSP блокирует - нужно настроить его правильно
- Или временно отключить для тестирования

### 3. Проблемы с сертификатом SSL

**Проблема:** Браузер считает соединение небезопасным.

**Проверка:**
1. В Network tab кликните на запрос к `claritybot.ru` (document)
2. Перейдите на вкладку **Security**
3. Проверьте, нет ли ошибок сертификата

### 4. CORS блокировка на уровне браузера

**Проблема:** Браузер блокирует запросы из-за CORS политики.

**Проверка:**
1. В Developer Tools перейдите на вкладку **Console**
2. Проверьте, есть ли ошибки:
   - "CORS policy: No 'Access-Control-Allow-Origin' header"
   - "Cross-Origin Request Blocked"
   - "Mixed Content"

### 5. Проблема с путями в index.html

**Проблема:** Неправильные пути могут вызывать проблемы.

**Проверка:**
1. Откройте `dist/index.html`
2. Проверьте пути к файлам:
   - Должны быть `/assets/index.BKhVh7uA.js` (с `/` в начале)
   - Не должны быть `assets/index.BKhVh7uA.js` (без `/`)

## Что проверить СЕЙЧАС:

### 1. Проверьте Console tab на ошибки

1. В Developer Tools перейдите на вкладку **Console**
2. Проверьте, есть ли ошибки (красные сообщения)
3. **Скопируйте все ошибки** - это покажет точную причину

### 2. Проверьте Security tab

1. В Network tab кликните на запрос к `claritybot.ru` (document)
2. Перейдите на вкладку **Security**
3. Проверьте, нет ли проблем с сертификатом

### 3. Проверьте Headers для главной страницы

1. В Network tab кликните на запрос к `claritybot.ru` (document)
2. Перейдите на вкладку **Headers**
3. Проверьте **Response Headers:**
   - Есть ли `Content-Security-Policy`?
   - Какой URL у запроса (HTTP или HTTPS)?

### 4. Попробуйте открыть файлы напрямую

Попробуйте открыть в новой вкладке:
- `https://claritybot.ru/assets/index.BKhVh7uA.js`
- `https://claritybot.ru/assets/index.CPWsFjWM.css`

**Результаты:**
- ✅ Если открываются → проблема в настройках загрузки на странице
- ❌ Если не открываются → Cloudflare блокирует или файлы недоступны

## Критично:

**Проверьте Console tab** - там должны быть ошибки, которые объясняют, почему браузер блокирует запросы. Скопируйте все ошибки из Console tab.
