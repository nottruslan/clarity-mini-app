# Срочное исправление белого экрана

## Проблема
Файлы `index.BKhVh7uA.js` и `index.CPWsFjWM.css` не загружаются с ошибкой "Network connection lost".

## Быстрое решение

### Вариант 1: Временно отключить прокси Cloudflare (для диагностики)

1. В Cloudflare Dashboard → DNS → Records
2. Для обеих записей (`claritybot.ru` и `www`) переключите оранжевое облако на **серое** (DNS only)
3. Подождите 2-3 минуты
4. Проверьте, открывается ли сайт напрямую через `clarityminiapp.netlify.app`
5. Если да - проблема в настройках Cloudflare прокси
6. Если нет - проблема в деплое Netlify

### Вариант 2: Проверка режима SSL/TLS (критично!)

1. В Cloudflare Dashboard → **SSL/TLS** → **Overview**
2. **ОБЯЗАТЕЛЬНО** должен быть режим **"Full"** (не "Flexible"!)
3. Если стоит "Flexible" - измените на "Full"
4. Это критично для работы через прокси!

### Вариант 3: Очистка кеша Cloudflare

1. В Cloudflare Dashboard → **Caching** → **Configuration**
2. Нажмите **"Purge Everything"**
3. Подтвердите очистку
4. Подождите 1-2 минуты
5. Обновите страницу в режиме инкогнито

### Вариант 4: Проверка деплоя Netlify

1. Зайдите в Netlify Dashboard
2. Проверьте последний деплой - должен быть статус "Published"
3. Если деплой не завершен или с ошибкой:
   - Пересоберите проект: `npm run build`
   - Задеплойте: `npm run deploy`
   - Или сделайте новый коммит и push в Git

### Вариант 5: Проверка через прямой доступ к Netlify

Попробуйте открыть напрямую:
- `https://clarityminiapp.netlify.app/assets/index.BKhVh7uA.js`
- `https://clarityminiapp.netlify.app/assets/index.CPWsFjWM.css`

Если файлы открываются напрямую через Netlify, но не через Cloudflare - проблема точно в настройках Cloudflare.

## Рекомендуемый порядок действий

1. **Сначала проверьте SSL/TLS режим** (Вариант 2) - это самая частая причина
2. **Очистите кеш Cloudflare** (Вариант 3)
3. **Проверьте деплой Netlify** (Вариант 4)
4. Если не помогло - временно отключите прокси (Вариант 1) для диагностики

## После исправления

1. Подождите 5-10 минут для распространения изменений
2. Откройте сайт в режиме инкогнито
3. Проверьте консоль браузера - ошибок быть не должно
4. Проверьте Network tab - файлы должны загружаться со статусом 200
