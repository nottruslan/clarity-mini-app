# Инструкция по тестированию Telegram Mini App

## Способ 1: Тестовый HTML файл (рекомендуется)

1. Убедитесь, что dev-сервер запущен:
```bash
npm run dev
```

2. Откройте файл `test-telegram-app.html` в браузере:
   - Двойной клик по файлу, или
   - Откройте через браузер: `file:///путь/к/проекту/test-telegram-app.html`

3. Проверьте консоль браузера (F12) на наличие ошибок

4. Что должно произойти:
   - Вверху появится зеленая полоса с сообщением об успешной загрузке
   - Приложение должно загрузиться в iframe
   - В консоли должны быть логи вызовов Telegram WebApp API

## Способ 2: Прямой доступ к dev-серверу

1. Запустите dev-сервер:
```bash
npm run dev
```

2. Откройте в браузере: `http://localhost:3000`

3. Приложение должно загрузиться, но некоторые функции Telegram API будут недоступны (fallback режим)

## Способ 3: Тестирование в реальном Telegram

1. Соберите проект:
```bash
npm run build
```

2. Загрузите папку `dist` на хостинг (Netlify, Vercel и т.д.)

3. Настройте Telegram бота через @BotFather:
   - Используйте команду `/newapp` или `/setmenubutton`
   - Укажите URL вашего приложения

4. Откройте мини-приложение через бота в Telegram

## Что проверять:

✅ Приложение загружается без ошибок
✅ Нет ошибок в консоли браузера
✅ Отображается главная страница с секциями
✅ Навигация работает корректно
✅ Нет ошибок при вызове `setHeaderColor`
✅ Нет ошибок при инициализации Telegram WebApp API

## Известные проблемы:

- В тестовом режиме (без Telegram) некоторые функции могут работать в fallback режиме
- Cloud Storage будет использовать localStorage как fallback
- Кнопка "Назад" может не работать вне Telegram
