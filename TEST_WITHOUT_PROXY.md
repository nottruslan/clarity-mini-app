1. Откройте `https://claritybot.ru/` в режиме инкогнито

1. Откройте Developer Tools (F12)
2. Перейдите на вкладку **Network**
3. Обновите страницу (F5)

## Результаты теста

### Если сайт РАБОТАЕТ без прокси:

✅ **Проблема в настройках Cloudflare прокси**

**Что сделать:**
1. Верните прокси обратно (оранжевое облако)
2. Проверьте следующие настройки Cloudflare:

**A. Speed Optimization:**
- Cloudflare Dashboard → **Speed** → **Optimization**
- **ОБЯЗАТЕЛЬНО отключите:**
  - Auto Minify
  - Rocket Loader

**B. WAF (Web Application Firewall):**
- Cloudflare Dashboard → **Security** → **WAF**
- Временно отключите все Managed Rules
- Проверьте Custom Rules - нет ли правил, блокирующих `/assets/*`

**C. Caching:**
- Cloudflare Dashboard → **Caching** → **Configuration**
- Установите **Caching Level** на "Standard"
- Очистите кеш: **Purge Everything**

**D. Page Rules:**
- Cloudflare Dashboard → **Rules** → **Page Rules**
- Проверьте, нет ли правил, блокирующих запросы

### Если сайт НЕ РАБОТАЕТ без прокси:

❌ **Проблема не в Cloudflare прокси**

**Возможные причины:**
1. Проблема с DNS (домен не резолвится)
2. Проблема с деплоем Netlify
3. Проблема с SSL сертификатом Netlify
4. Проблема в коде проекта

**Что проверить:**
1. Откройте напрямую: `https://clarityminiapp.netlify.app/`
   - Если работает → проблема с DNS или кастомным доменом
   - Если не работает → проблема в деплое Netlify

2. Проверьте логи деплоя в Netlify:
   - Netlify Dashboard → **Deploys** → последний деплой
   - Проверьте статус и ошибки

3. Проверьте DNS записи:
   - Убедитесь, что DNS записи правильные
   - Подождите 10-15 минут для распространения DNS

## Важно

После отключения прокси подождите **2-3 минуты** для распространения DNS изменений перед проверкой.
